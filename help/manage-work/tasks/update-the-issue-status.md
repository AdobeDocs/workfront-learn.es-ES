# Actualizar estados de problemas de forma eficaz

En este vídeo aprenderá sobre los estados de problemas y las prácticas recomendadas para utilizarlos.

>[!VIDEO](https://video.tv.adobe.com/v/3472962/?quality=12&learn=on&enablevpops=1)

## Principales conclusiones

* **¿En qué se diferencian los problemas de las tareas?** Los problemas son diferentes de las tareas en el sentido de que un problema representa trabajo no planificado. Si un problema debe incluirse como trabajo planificado, debe convertirse en una tarea o un proyecto.
* **¿Por qué es importante revisar rápidamente los problemas de un proyecto?** Cuando hay problemas pendientes en un proyecto, representan un riesgo potencial para la programación o el presupuesto del proyecto y deben asignarse rápidamente a alguien que pueda investigarlos y tomar o recomendar las medidas adecuadas.
* **¿Qué sucede con los problemas de una cola de solicitudes?:** En una cola de solicitudes, los problemas se pueden usar más como elementos de listas de tareas pendientes. La necesidad de convertirlos en una tarea o proyecto puede ser diferente, pero la práctica de actualizar el estado de forma oportuna sigue siendo muy importante.
