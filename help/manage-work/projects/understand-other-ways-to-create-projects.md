# Explore cuatro métodos para crear un proyecto

El vídeo proporciona una guía completa sobre la creación de proyectos en Workfront mediante varios métodos:

**Creando un proyecto a partir de una plantilla:**

* Vaya a la página Proyectos, haga clic en &quot;Nuevo proyecto&quot; y seleccione &quot;Nuevo proyecto a partir de la plantilla&quot;. &#x200B;
* Examine las plantillas activas, obtenga una vista previa de los resúmenes (por ejemplo, duración, propietario, resumen de tareas, formularios personalizados) y seleccione una plantilla. &#x200B;
* Se puede acceder directamente a las plantillas favoritas para realizar una selección más rápida. &#x200B;
* Rellene la información inicial (por ejemplo, el nombre del proyecto) y haga clic en &quot;Crear proyecto&quot;. Los campos se pueden editar más adelante. 

**Creando un proyecto desde cero:**

* Vaya a la página Proyectos, haga clic en &quot;Nuevo proyecto&quot; y seleccione &quot;Nuevo proyecto&quot;. &#x200B;
* Asigne un nombre al proyecto y edite detalles como las opciones de uso compartido, los datos personalizados, el portafolio y el programa. &#x200B;
* Los valores predeterminados establecidos por el administrador del sistema o del grupo se pueden aceptar o cambiar. &#x200B;
* Guarde el proyecto, que inicialmente solo se comparte con el creador y los administradores del sistema. &#x200B;
* Las tareas se pueden agregar escribiendo nombres, con campos como duración y horas planificadas editables. &#x200B;

**Copiando un proyecto:**

* Abra un proyecto existente, haga clic en el menú de tres puntos y seleccione Copiar. &#x200B;
* Cambie el nombre y el estado (por ejemplo, Planificación) y anule la selección de los elementos que no desea copiar (por ejemplo, gastos). &#x200B;
* Editar detalles como la fecha planificada de inicio después de la copia. &#x200B;

**Importando un archivo de proyecto de Microsoft:**

* Guarde el archivo de MS Project como un archivo XML. &#x200B;
* Utilice la opción &quot;Importar MS Project&quot; en el menú Nuevo proyecto, seleccione el archivo y comience la importación. &#x200B;
* Algunos ajustes de MS Project (por ejemplo, comentarios, formularios personalizados, hitos) no se transferirán. &#x200B;
* Establezca el estado en Planificación para revisar y editar el proyecto importado antes de que se ponga en marcha. &#x200B;


El vídeo destaca la flexibilidad de Workfront para crear proyectos a través de plantillas, arañar, copiar o importar, para satisfacer diferentes necesidades del usuario. &#x200B;

## Principales conclusiones

* **Las plantillas simplifican la creación de proyectos:**
* Se recomienda encarecidamente utilizar una plantilla, ya que puede predefinir detalles clave como tareas, formularios y opciones de uso compartido, lo que ahorra tiempo y esfuerzo. &#x200B;
* **Plantillas favoritas para acceso rápido:**
* Las plantillas favoritas permiten a los usuarios evitar el proceso de selección y crear directamente un proyecto, lo que optimiza el flujo de trabajo. &#x200B;
* **Crear proyectos desde cero ofrece flexibilidad:**
* Aunque es más manual, la creación de un proyecto desde cero permite la personalización de todos los detalles, ideal para proyectos simples o privados. &#x200B;
* **Copiar proyectos ahorra tiempo:**
* Al copiar un proyecto existente, se replican su estructura y configuración, con opciones para modificar o excluir elementos específicos. &#x200B;
* **Importando archivos de MS Project para la transición:**
* Workfront admite la importación de archivos de MS Project (guardados como XML), lo que permite a los usuarios realizar la transición a proyectos en curso, aunque algunos datos (por ejemplo, comentarios, hitos) no se transferirán. &#x200B;
